    QuOptuna Next - Quick Start Guide üöÄ

    Get up and running with QuOptuna Next in under 5 minutes!

    üéØ What You'll Get

    A modern, drag-and-drop interface for building quantum ML workflows with real-time optimization tracking.

    üìã Prerequisites

    Choose one: - Docker (easiest) - Just Docker and docker-compose - Local Development - Node.js 18+ and Python 3.10+

    üöÄ Option 1: Docker (Recommended)

    Step 1: Start the Services

    cd /path/to/quoptuna
docker-compose up

    That's it! üéâ

    Step 2: Access the Application

    • Frontend: http://localhost:5173
    • Backend API: http://localhost:8000
    • API Docs: http://localhost:8000/api/docs

    üíª Option 2: Local Development

    Step 1: Start Backend

    cd backend

# Create virtual environment
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# Install dependencies
pip install -e .

# Start server
uvicorn app.main:app --reload

    Backend running at http://localhost:8000 ‚úì

    Step 2: Start Frontend

    # New terminal
cd frontend

# Install dependencies
npm install

# Start dev server
npm run dev

    Frontend running at http://localhost:5173 ‚úì

    üé® Using the Application

    1. Create Your First Workflow

    1. Navigate to Workflow Builder
    2. Drag nodes from the left palette onto the canvas
    3. Connect nodes by dragging from output (bottom) to input (top) handles

    Example Workflow: 

    Upload CSV ‚Üí Feature Selection ‚Üí Train/Test Split ‚Üí Quantum Model ‚Üí Optimization ‚Üí SHAP Analysis

    2. Configure Nodes

    Click any node to configure its parameters: - Dataset selection - Model type - Hyperparameter ranges - Analysis options

    3. Run the Workflow

    Click Run in the toolbar to execute your pipeline!

    üîß Development Commands

    Frontend

    cd frontend

# Development
npm run dev          # Start dev server
npm run build        # Build for production
npm run preview      # Preview production build
npm run type-check   # TypeScript check

# Clean install
rm -rf node_modules package-lock.json && npm install

    Backend

    cd backend

# Development
uvicorn app.main:app --reload        # Start with auto-reload
python -m pytest                      # Run tests
ruff check .                          # Lint code
ruff format .                         # Format code

# Clean environment
rm -rf .venv && python -m venv .venv

    Docker

    # Start services
docker-compose up              # Foreground
docker-compose up -d           # Background

# View logs
docker-compose logs -f         # All services
docker-compose logs -f frontend  # Frontend only
docker-compose logs -f backend   # Backend only

# Stop and clean
docker-compose down            # Stop services
docker-compose down -v         # Stop and remove volumes
docker-compose build           # Rebuild images

    üêõ Troubleshooting

    Port Already in Use?

    # Change frontend port
cd frontend
# Edit vite.config.ts: server: { port: 3000 }

# Change backend port
cd backend
# Start with: uvicorn app.main:app --port 8001

    Dependencies Not Installing?

    # Frontend
cd frontend
rm -rf node_modules package-lock.json
npm cache clean --force
npm install

# Backend
cd backend
rm -rf .venv
python -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -e .

    Docker Issues?

    # Full clean restart
docker-compose down -v
docker system prune -a
docker-compose build --no-cache
docker-compose up

    üéì Tutorial: Your First Quantum ML Workflow

    Goal: Train a quantum model on the Iris dataset

    Time: 5 minutes

    1. Upload Data
    2. Go to Data Explorer
    3. Click "UCI Repository"

    4. Select "Iris Dataset"

    5. Build Workflow

    6. Go to Workflow Builder
    7. Drag: UCI Dataset ‚Üí Feature Selection ‚Üí Train/Test Split ‚Üí Standard Scaler ‚Üí Quantum Model ‚Üí Optuna Config ‚Üí SHAP Analysis

    8. Connect all nodes

    9. Configure

    10. UCI Dataset: Select "iris"
    11. Feature Selection: X = [sepal_length, sepal_width, petal_length, petal_width], y = species
    12. Quantum Model: Type = "Data Reuploading"

    13. Optuna Config: Trials = 20

    14. Execute

    15. Click Run
    16. Watch real-time progress

    17. View results in Analytics

    18. Analyze

    19. Go to Analytics
    20. See SHAP plots
    21. Generate AI report
